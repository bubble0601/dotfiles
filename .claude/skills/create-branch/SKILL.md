# ブランチの作成

## ユーザー入力

```
$ARGUMENTS
```

処理を進める前に、ユーザー入力を必ず考慮してください(空でない場合)。
ユーザー入力が与えられていない場合はユーザーにタスクの概要を尋ねてください

## ブランチを作成する

1. 簡潔な短縮名(2~4 単語)を生成する

- 機能説明を分析し、最も意味のあるキーワードを抽出
- 機能の本質を捉えた英語で 2〜4 単語の短縮名を作成
- 技術用語や略語を保持(OAuth2、API、JWT など)
- 簡潔でありながら、一目で機能を理解できる程度に説明的に

2. 考えたブランチ名にタスクの性質に応じて新機能なら`feature/`、バグ修正なら`bugfix/`、リファクタなら`refactor/`の prefix をつけてブランチ名とする

3. 新しいブランチを作成する前に既存ブランチを確認:
   a. まず、最新情報を確実に取得するために、すべてのリモートブランチをフェッチ:

   ```bash
    git fetch --all --prune
   ```

   b. ブランチ一覧を検索し考えたブランチ名が既に存在する場合は 1 に戻って別のブランチ名を考え直す

   ```bash
   git branch -a | grep [ブランチ名]
   ```

4. ブランチを作成する
   git switch サブコマンドを使用して
   ベースブランチは指定がなければ`dev`
   dev をベースにする場合は git fetch で dev を最新の状態にする
