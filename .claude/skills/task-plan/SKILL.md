# タスクの実装を計画

このコマンドが呼び出されたら以下の指示に従って順番に進めてください
plan サブエージェントを使用してください

## Step1: タスクのディレクトリを特定する

まずは`*/tasks`にマッチするタスクを格納するディレクトリを特定してください。なければ root 配下に tasks ディレクトリを作成してください。

```bash
find -L . -type d -name "tasks" -not -path "*/node_modules/*" -not -path "*/.git/*"
```

このディレクトリを以下`TASK_ROOT_DIR`とします

**引数が指定されている場合:**
TASK_ROOT_DIR を検索し、`$(printf "%03d" $1)`で始まるディレクトリを見つけてください

**引数が指定されていない場合:**
TASK_ROOT_DIR を検索し、ディレクトリのうち先頭 3 文字を数値にしたときに最大のものを見つけてください

## spec.md の読み込み

特定したディレクトリの中にある spec.md を読み、要件を理解してください

### spec.md が見つからない場合

task-init コマンドを実行するようユーザーに指示してください
spec.md が見つからなかった場合はこのコマンドはここで終了し以降の指示は無視してください

### spec.md が見つかった場合

1. spec.md に記載されているブランチ名を確認してください
   - ブランチ名が記載されている場合、現在のブランチと一致しているか確認してください
   - 一致していない場合は、ユーザーに警告を出してください

## Step2: 計画

まだ実装は始めないこと(plan.md は編集してよい)

### Phase1: 既存の plan.md の確認

特定したタスクディレクトリ内に plan.md が既に存在するか確認してください

- 存在する場合：ユーザーに上書きするか確認してください
- 存在しない場合：次のステップに進んでください

### Phase2: 対話的に実装計画策定

spec.md の要件を実装に落とし込むため、以下を行ってください：

**実施内容:**

- spec.md の機能要件に関連するファイル、関数、データモデル、API を探索
- 既存コードへの影響範囲を分析
- 実装に必要な技術スタックやライブラリを特定

**実装プランの作成:**
Tasks機能を使いTaskに分解してください。

**ガイドライン:**

- 不明点があれば**AskUserQuestion**ツールを使って不明点がなくなるまでユーザーに質問し明確化してください
- [重要]まだ実装は開始しないでください

### Phase3: タスクIDをspec.mdに保存

spec.mdの先頭の作成日の次にタスクIDを書き込んで

## ユーザーへ次のステップの指示

実装プランを確認して、問題なければ task-implement コマンドを実行するようにユーザーに促してください
