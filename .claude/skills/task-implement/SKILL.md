# タスクを実装

このコマンドが呼び出されたら以下の指示に従って順番に進めてください

## タスクのディレクトリを特定する

まずは`*/tasks`にマッチするタスクを格納するディレクトリを特定してください。なければ root 配下に tasks ディレクトリを作成してください。

```bash
find -L . -type d -name "tasks" -not -path "*/node_modules/*" -not -path "*/.git/*"
```

このディレクトリを以下`TASK_ROOT_DIR`とします

**引数が指定されている場合:**
TASK_ROOT_DIR を検索し、`$(printf "%03d" $1)`で始まるディレクトリを見つけてください

**引数が指定されていない場合:**
TASK_ROOT_DIR を検索し、ディレクトリのうち先頭 3 文字を数値にしたときに最大のものを見つけてください

## spec.md の読み込み

特定したディレクトリの中にある spec.md を読み、要件を理解してください

### spec.md が見つからない場合

task-explore コマンドを実行するようユーザーに指示してください
spec.md が見つからなかった場合はこのコマンドはここで終了し以降の指示は無視してください

### spec.md が見つかった場合

1. spec.md に記載されているブランチ名を確認してください
   - ブランチ名が記載されている場合、現在のブランチと一致しているか確認してください
   - 一致していない場合は、ユーザーに警告を出してください

## plan.md の読み込み

特定したディレクトリの中にある plan.md を読み、実装計画を理解してください

### plan.md が見つからない場合

**EnterPlanMode**ツールを使用して plan モードに入ってください。
実装計画が決まったらそれを元に以下の手順で実装を開始してください。

## 実装

### 実装プロセス

実装計画に従い、以下を遵守して実装してください：

1. **実装ステップの順守**
   - 実装ステップの順序に従って実装
   - (plan.md がある場合)各ステップを完了したら、plan.md のチェックボックスにチェックを入れる

#### ガイドライン

- 不明点や技術的な判断が必要な場合は、都度**AskUserQuestion**ツールを使ってユーザーに質問
- 複数の実装方法が考えられる場合も 都度**AskUserQuestion**ツールで選択肢を提示してユーザーに確認

### 実装後の検証

実装完了後、以下を実行してください：

1. **型チェック**
   - プロジェクトで利用可能な型チェックコマンドを実行
   - 型エラーがある場合は修正

### 完了条件

以下がすべて満たされた場合、実装完了とみなします：

- plan の全実装ステップが完了している
- 型チェックがエラーなく通る
- ビルドが成功する
