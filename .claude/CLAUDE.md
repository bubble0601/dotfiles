## 出力スタイル(視覚階層ルール)
  ターミナルは字サイズを変えられないため、**罫線・記号・余白**で iOS アプリ風の3階層を作る。長めの応答・レビュー・レポートに適用し、短い一問一答やmarkdownには使わない。
  ## レイヤー定義
  ### L1: タイトル(応答全体の主題 / 1応答につき1つまで)
  `┏━` 角付きの上下バーで挟み、タイトルは **中央寄せ**。センタリングは NBSP(U+00A0、半角スペースでは4つ以上だと code block 化する)で左パディングを作る。バー幅は 80
  文字固定(角 `┏┓` `┗┛` + `━` 78本)。
  書式:
  ```
  ┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
                                    **タイトル**
  ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
  ```
  中央寄せの計算: 表示幅 = ASCII 1 + 日本語 2。パディング幅 = (78 - 表示幅) / 2。パディングは NBSP で埋める。ズレが±1文字出ても可。
  ### L2: セクション(章見出し)
  `═` 二重線 80 本で上下を挟み、中に中央寄せの太字テキスト。記号は付けない。
  書式:
  ```
  ════════════════════════════════════════════════════════════════════════════════
                                   **セクション名**
  ════════════════════════════════════════════════════════════════════════════════
  ```
  ### L3: サブセクション(小見出し)
  `▎` 縦バー + 太字。装飾は最小限。**必ず前後に空行を1行ずつ入れる**(入れないと renderer が上下の間隔を不揃いに詰めることがある)。
  書式:
  ```
  ▎**サブ見出し**
  本文がすぐ下に続く。
  ```
